🎮 Java 魔物獵人（Monster Hunter）模擬系統這是一個基於 Java 物件導向程式設計 (OOP) 開發的文字冒險 RPG 系統。玩家扮演獵人，透過挑戰不同星級的魔物獲得經驗與金錢，並在商店中強化武器與補給，最終挑戰高難度的強大魔物。🌟 系統特色1. ⚔️ 策略性回合戰鬥動態決策：玩家每回合可選擇「攻擊」或「喝水」。減傷機制：選擇喝水回合雖無法攻擊，但可回復 HP 並獲得 50% 傷害減免。戰後休整：成功狩獵後，系統會自動回復玩家剩餘血量的 50%，支援連續作戰。2. ⚒️ 裝備與強化系統無限強化：商店提供武器鍛造功能，強化後武器名稱會動態標示 (+1)、(+2) 並永久提升攻擊力。階層商店：提供大小兩種回復藥水，玩家需管理金錢資源進行補給。3. 📜 多樣化任務板開放式挑戰：打破線性解鎖，全星級任務初始開放，玩家可自由選擇挑戰對象。動態生成：每個星級包含 2-3 種特色魔物（如：一星青熊獸、三星轟龍），具備不同的數值落差與經驗回報。4. 💾 存檔紀錄功能完整持久化：支援 File IO，可保存等級、經驗、金錢、藥水數量及強化後的武器狀態。🏗️ 系統架構 (System Architecture)本專案採用分層架構設計，確保程式碼具備高可維護性與擴展性：model (資料實體)：定義 Character（基類）、Player、Monster、Quest、Weapon 等核心物件。service (邏輯處理)：BattleService：控制戰鬥流程、傷害計算與減傷邏輯。StoreService：處理交易、藥水發放與武器字串解析強化。SaveService：處理檔案讀寫與資料序列化。QuestService：管理任務清單與看板顯示。factory (物件工廠)：MonsterFactory 負責根據星級與編號產出特定的魔物實體。enumtype (列舉值)：定義 QuestRank (星等) 與 QuestStatus (狀態)。🛠️ 技術亮點 (Technical Highlights)OOP 三大特性：繼承：Player 與 Monster 繼承自抽象類別 Character，共用屬性與受傷邏輯。封裝：所有屬性皆為 private，透過 Getter/Setter 存取。多型：重寫 attack() 方法以實現玩家與怪物不同的攻擊行為。工廠模式 (Factory Pattern)：透過 MonsterFactory 封裝魔物生成過程，隱藏物件創建細節。字串解析技術：武器強化時，利用 lastIndexOf 與 substring 解析名稱中的強化等級並動態更新。File IO 應用：實作 BufferedReader 與 BufferedWriter 進行遊戲數據的存取。📝 如何運行確保環境已安裝 JDK 17 或以上版本。將所有原始碼放入對應的 Package 資料夾中（model, service, factory, app）。編譯並執行 app.Main。遊戲進度將自動儲存於同目錄下的 save.txt 中。📊 遊戲數值平衡預覽難度代表魔物HP攻擊力經驗/獎勵1 ★青熊獸605302 ★雌火龍150181203 ★碎龍45055450專案開發者：[你的名字]版本：v1.0 (作品集最終版)
